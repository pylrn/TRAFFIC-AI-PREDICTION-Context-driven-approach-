📊 How Each Algorithm Works

Four AI models run in parallel on every prediction. Each one looks at traffic differently. Tags marked 📚 In Your Syllabus show where these ideas appear directly in your course.

🔀
Markov Chain
"What happens next depends only on what’s happening right now."

Imagine traffic has 4 states: Free, Moderate, Heavy, Jam. A Markov Chain is just a table of probabilities that says “if traffic is X now, how likely is it to be Y in the next minute?” That’s it — no memory of yesterday, just the current moment.

The core idea — Conditional Probability (Module 2, Topic 7)
P(next state = Jam | current state = Heavy) = 0.20 P(next state = Heavy | current state = Heavy) = 0.55 → These are just conditional probabilities from your syllabus!

The full table is a 4×4 transition matrix. Each row is a Probability Mass Function (sums to 1.0) over the four possible next states. At runtime, weights shift if speed is low or weather is bad.

Transition matrix (each row = a PMF — Module 2, Topic 2)

💡 Concepts Used
  • Random Variables — M2 Topic 1 — Markov Chain represents each traffic condition (free/moderate/heavy/jam) as a value of a discrete random variable.
  • Probability Mass Function — M2 Topic 2 — Each row of the Markov Chain transition matrix is a PMF listing the probability of moving to every possible next state.
  • Conditional Probability — M2 Topic 7 — The Markov Chain's core rule is P(next | current) — a conditional distribution computed at every time step.
  • Marginal Probability — M2 Topic 6 — The Markov Chain's long-run steady-state is found by computing the marginal distribution via repeated matrix multiplication.
  • Discrete Distributions — M4 Topic 1 — Every prediction Markov Chain makes is a discrete PMF assigning a probability to each of the 4 traffic states.

Random Forest
"150 decision trees vote — majority wins."

A Random Forest grows 150 decision trees, each trained on a slightly different random sample of data. Each tree looks at speed, vehicle count, time, weather, and accident flag, then votes on the traffic class. The final probability = fraction of trees that voted for each class. More agreement = higher confidence.

Probability = vote fraction (relates to PMF — Module 2, Topic 2)
P(class = Jam | inputs) = (trees that voted Jam) / 150 e.g. 102 out of 150 trees say Jam → P(Jam) = 0.68

Each tree splits data using Gini Impurity — a measure of how “mixed” a group is. This is mathematically identical to variance/dispersion from your stats module — lower Gini = more pure = a good split.

Gini Impurity — same idea as Dispersion (Module 1, Topic 3)
Gini = 1 − Σ p_c² (0 = perfectly pure, 0.75 = max mixed) If a node is all-Jam: Gini = 1 − 1² = 0 ← best possible split

💡 Concepts Used
  • Measures of Dispersion — M1 Topic 3 — Random Forest splits nodes using Gini Impurity, a measure of label spread that is mathematically identical to variance.
  • Conditional Probability — M2 Topic 7 — Random Forest outputs P(class | inputs) — a conditional distribution over traffic states given the feature values.
  • Mathematical Expectation — M2 Topic 8 — The forest's final probability is the expected class across 150 tree votes, a direct application of expectation.
  • Correlation & Feature Importance — M3 Topic 2 — Random Forest ranks which features (speed, density, weather) most correlate with the traffic outcome.
  • Analysis of Variance — M6 Topics 7–9 — Each Random Forest split minimises within-group variance — the exact same goal as ANOVA.
  • Chi-Square Independence — M6 Topic 5 — Random Forest implicitly tests whether input features are independent of the output class, mirroring the chi-square test.

LSTM Neural Network
"It remembers the last 6 minutes of traffic and spots patterns over time."

An LSTM is a neural network with a built-in memory cell. It reads the last 6 timesteps of traffic data in order and learns patterns like “density has been climbing for 6 straight minutes → jam is coming.” That's something a snapshot model simply cannot do.

Output = Softmax — turns raw scores into a probability distribution (Module 2, Topic 5 / Module 4)
P(class = c | sequence) = exp(score_c) / Σ exp(score_j) → all 4 probabilities sum to 1.0 (a valid PMF over traffic states) → same as a discrete probability distribution from Module 4

The network has four gates (forget, input, cell, output) that decide what to remember and what to discard at each step. Think of them as a smart highlighter that keeps the important trend and ignores noise.

Forget gate — decides what old info to erase
f_t = σ( weights · [last_hidden, current_input] ) σ = sigmoid → output is always between 0 and 1 0 = forget everything, 1 = keep everything

💡 Concepts Used
  • Random Variables — M2 Topic 1 — Each of the 6 timestep inputs to LSTM (speed, count, etc.) is a random variable sampled from traffic at that moment.
  • Joint Distributions — M2 Topics 4–5 — The 6-step window is a joint distribution over 6 dependent random variables — LSTM learns this joint structure end-to-end.
  • Mathematical Expectation — M2 Topic 8 — The LSTM hidden state is a weighted expectation over past inputs, continuously refined through training.
  • Normal Distribution — M4 Topic 4 — LSTM weights are initialised from a Normal distribution; gradient descent shifts them toward optimal values.
  • Discrete PMF — M4 Topic 1 — The Softmax layer converts LSTM's raw output scores into a discrete PMF over all 4 traffic states.
  • Hypothesis Testing — M5 Topic 1 — Training LSTM is iterative hypothesis testing: each update checks whether new weights better explain the observed traffic data.

Bayesian Network
"Uses Bayes’ Theorem to reason about cause and effect."

A Bayesian Network is a graph where arrows mean “this causes that.” Here: Weather, Time of Day, and Vehicle Density all cause Congestion. For each combination of inputs it stores a lookup table of probabilities (a Conditional Probability Table). This is literally Bayes’ Theorem from your syllabus applied to traffic.

Bayes’ Theorem in action (Module 2, Topic 7)
P(Congestion | Weather=rain, Time=rush, Density=high) = P(rain, rush, high | Congestion) × P(Congestion) ÷ P(rain, rush, high) → We read the answer directly from the CPT lookup table

The graph also uses Joint Probability: the full joint distribution over all four variables factors neatly along the graph arrows. No need to track every possible combination — the edges tell you which variables are independent of each other.

Joint distribution factorisation (Module 2, Topics 4–6)
P(Weather, Time, Density, Congestion) = P(Weather) × P(Time) × P(Density) × P(Congestion | W, T, D) → Joint PMF factored using conditional independence

💡 Concepts Used
  • Conditional Probability — M2 Topic 7 — The Bayesian Network is entirely built from Conditional Probability Tables (CPTs): P(Congestion | Weather, Time, Density).
  • Joint PMF — M2 Topic 4 — The Bayesian Network stores the full joint distribution P(Weather, Time, Density, Congestion) factored along the graph arrows.
  • Marginal Distributions — M2 Topic 6 — The output P(Congestion) is a marginal distribution obtained by summing out Weather, Time, and Density via Variable Elimination.
  • Mathematical Expectation — M2 Topic 8 — The Bayesian Network computes expected congestion by weighting each state's probability with its corresponding value.
  • Discrete Distributions — M4 Topic 1 — Every node in the Bayesian Network is a discrete distribution; inference produces a discrete PMF over traffic states.
  • Reliability — M7 — The Bayesian Network's conditional independence structure mirrors fault-tree analysis in system reliability studied in Module 7.

✨ Why combine all four?  Each model is strong in different contexts. The ensemble picks context-specific weights and computes:

P_final = Σ_m w_m · P_m  where Σ w_m = 1.0

This is a convex combination of probability distributions — the result is itself a valid PMF, giving calibrated confidence across all four traffic states. This is Mathematical Expectation (Module 2, Topic 8) applied to model outputs.
